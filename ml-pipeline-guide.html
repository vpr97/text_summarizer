summarizer.py — ML Model Logic

PYTHONapp/summarizer.py
from transformers import pipeline

# Load model once at startup (facebook/bart-large-cnn)
summarizer = pipeline(
    "summarization",
    model="facebook/bart-large-cnn"
)

def summarize_text(text: str, max_length: int = 130, min_length: int = 30) -> str:
    """Summarize input text using BART model."""
    if len(text.split()) < 50:
        raise ValueError("Text too short. Provide at least 50 words.")

    result = summarizer(
        text,
        max_length=max_length,
        min_length=min_length,
        do_sample=False
    )
    return result[0]["summary_text"]

main.py — FastAPI Backend

PYTHONapp/main.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from app.summarizer import summarize_text

app = FastAPI(title="Text Summarizer API", version="1.0")

class SummarizeRequest(BaseModel):
    text: str
    max_length: int = 130
    min_length: int = 30

class SummarizeResponse(BaseModel):
    summary: str

@app.get("/")
def health_check():
    return {"status": "running"}

@app.post("/summarize", response_model=SummarizeResponse)
def summarize(request: SummarizeRequest):
    try:
        summary = summarize_text(
            request.text, request.max_length, request.min_length
        )
        return SummarizeResponse(summary=summary)
    except ValueError as e:
        raise HTTPException(status_code=400, detail=str(e))

streamlit_app.py — Frontend UI

PYTHONstreamlit_app.py
import streamlit as st
import requests

st.set_page_config(page_title="Text Summarizer", page_icon="📝")
st.title("📝 AI Text Summarizer")
st.markdown("Paste any long text and get a concise AI summary.")

API_URL = "http://localhost:8000/summarize"  # points to FastAPI

text_input = st.text_area("Input Text", height=250,
    placeholder="Paste a long article or paragraph here...")

col1, col2 = st.columns(2)
max_len = col1.slider("Max Summary Length", 50, 300, 130)
min_len = col2.slider("Min Summary Length", 10, 100, 30)

if st.button("✨ Summarize", type="primary"):
    if text_input.strip():
        with st.spinner("Summarizing..."):
            response = requests.post(API_URL, json={
                "text": text_input,
                "max_length": max_len,
                "min_length": min_len
            })
        if response.status_code == 200:
            st.success("Summary")
            st.write(response.json()["summary"])
        else:
            st.error(response.json()["detail"])
    else:
        st.warning("Please enter some text first.")

Push Docker Image to AWS ECR
AWS CLI · create ECR repo · tag · push

AWS ECR (Elastic Container Registry) is the private Docker registry on AWS. You'll push your image there so AWS can deploy it.

Prerequisites

  • Install AWS CLI: brew install awscli or download from aws.amazon.com
  • Run aws configure and enter your Access Key ID, Secret Key, region (us-east-1)

Create ECR Repository & Push

BASHterminal
# Set your AWS account ID and region
export AWS_ACCOUNT_ID=123456789012   # replace with yours
export AWS_REGION=us-east-1
export REPO_NAME=text-summarizer

# Step 1: Create ECR repository
aws ecr create-repository \
  --repository-name $REPO_NAME \
  --region $AWS_REGION

# Step 2: Authenticate Docker to ECR
aws ecr get-login-password --region $AWS_REGION | \
  docker login --username AWS \
  --password-stdin $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com

# Step 3: Tag your local image
docker tag text-summarizer:latest \
  $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/$REPO_NAME:latest

# Step 4: Push to ECR
docker push \
  $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/$REPO_NAME:latest
ℹ️ After pushing, go to AWS Console → ECR and you'll see your image listed under the repository.

Deploy on AWS App Runner
Create service · configure ports · live URL

AWS App Runner is the simplest way to deploy a containerized app — no EC2 or Kubernetes needed.

Deploy via AWS CLI

BASHterminal
# Create App Runner service
aws apprunner create-service \
  --service-name text-summarizer-service \
  --source-configuration '{
    "ImageRepository": {
      "ImageIdentifier": "'$AWS_ACCOUNT_ID'.dkr.ecr.'$AWS_REGION'.amazonaws.com/text-summarizer:latest",
      "ImageRepositoryType": "ECR",
      "ImageConfiguration": {
        "Port": "8501"
      }
    },
    "AutoDeploymentsEnabled": true,
    "AuthenticationConfiguration": {
      "AccessRoleArn": "arn:aws:iam::'$AWS_ACCOUNT_ID':role/AppRunnerECRAccessRole"
    }
  }' \
  --instance-configuration '{"Cpu": "1 vCPU", "Memory": "2 GB"}' \
  --region $AWS_REGION

Or use the AWS Console (Easier)

  • Go to AWS Console → App Runner → Create Service
  • Source: Container registry → Amazon ECR
  • Select your text-summarizer:latest image
  • Port: 8501 (Streamlit)
  • CPU: 1 vCPU, Memory: 2 GB
  • Click Create & Deploy
  • In ~3 minutes you'll have a live https://xxx.awsapprunner.com URL
🎯 Auto-deploy: Enable Automatic deployments in App Runner — every time you push a new image to ECR, it will redeploy automatically!

Bonus: GitHub Actions CI/CD
Auto-build and push to ECR on every git push

Set up a workflow so every push to main automatically builds and pushes your Docker image to ECR.

YAML.github/workflows/deploy.yml
name: Deploy to AWS ECR

on:
  push:
    branches: [main]

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Configure AWS Credentials
        uses: aws-actions/configure-aws-credentials@v2
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: us-east-1

      - name: Login to ECR
        uses: aws-actions/amazon-ecr-login@v1

      - name: Build, Tag & Push
        run: |
          docker build -t text-summarizer .
          docker tag text-summarizer:latest \
            ${{ secrets.AWS_ACCOUNT_ID }}.dkr.ecr.us-east-1.amazonaws.com/text-summarizer:latest
          docker push \
            ${{ secrets.AWS_ACCOUNT_ID }}.dkr.ecr.us-east-1.amazonaws.com/text-summarizer:latest
🔐 Add AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, and AWS_ACCOUNT_ID as secrets in your GitHub repo under Settings → Secrets → Actions.
